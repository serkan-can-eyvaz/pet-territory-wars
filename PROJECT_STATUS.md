\# Agent Update Rules



At the end of \*\*every completed task\*\*, the AI agent MUST update this file.



The following sections are mandatory:



\- Current Task

\- Next Task

\- Active Branch

\- Milestone Progress

\- Repository Health

\- Completed Checklist

\- Blockers

\- Known Technical Debt (if introduced)



If none of these values changed, they must still be verified before reporting completion.



\---



\# Project Status Update Workflow



Every completed task must follow this exact order:



1\. Complete implementation.

2\. Run formatting.

3\. Run lint.

4\. Run tests.

5\. Verify build.

6\. Update PROJECT\_STATUS.md.

7\. Generate Completion Report.

8\. Stop.



A task is \*\*NOT\*\* considered complete until this file has been updated.



\---



\# Agent Restrictions



The AI agent MUST NOT:



\- Skip completed task updates.

\- Mark unfinished work as completed.

\- Reorder milestones.

\- Reorder tasks.

\- Delete completed checklist items.

\- Modify architecture versions.

\- Invent new milestones.

\- Add new tasks unless explicitly instructed.

\- Remove blockers without solving them.

\- Hide technical debt.



\---



\# Definition of Done



A task is considered completed only if:



\- Implementation is finished.

\- Architecture rules are respected.

\- Tests were added (when applicable).

\- All tests pass.

\- Build succeeds.

\- Lint succeeds.

\- Formatting succeeds.

\- Documentation is updated (if required).

\- PROJECT\_STATUS.md has been updated.

\- Completion Report has been generated.



Otherwise, the task must remain unchecked.
